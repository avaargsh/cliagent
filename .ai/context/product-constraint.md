# 产品约束

## 目标用户

- 主要用户：平台管理员、集成开发者、QA / 运维人员
- 次要用户：需要调试数字员工执行过程的业务管理员

## 场景边界

- 适用场景：企业数字员工任务执行、审批管控、工具调用审计、执行回放与诊断
- 不适用场景：通用聊天助手、IDE 编码助手、无人值守软件工厂

## 产品定位

- 主业务入口优先走 API、Webhook、企业 IM
- CLI（`dectl`）是"控制面"和"运维面"，负责创建工作单、审批、观察、回放和诊断
- 以 `work-order` 为一等业务对象，全生命周期可追踪

## MVP 约束

- 首版必须解决的问题：工作单驱动的单员工执行主链路（创建、运行、审批、观察、回放、诊断）
- 首版明确不做：通用多 Agent swarm、REST 全量端到端、数据库持久化、Windows 支持

## 验收偏好

- 输出语言：`zh-CN`
- 是否优先脚本化：`是`
- 是否允许交互依赖：`仅 TTY`
- 错误协议：稳定 exit code + 结构化 JSON 错误包裹
